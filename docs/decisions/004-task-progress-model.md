# D-004. Модель прогресса задачи

Статус: утверждено.

## Решение

Процент готовности задачи хранится отдельно от статуса и изменяется пользователем вручную.

Правила:

- диапазон значения: от `0%` до `100%`;
- значение по умолчанию при создании задачи: `0%`;
- изменение процента фиксируется в журнале истории;
- процент доступен через API;
- процент участвует в фильтрах и отчётах;
- каждый тип задачи имеет настройку `Использовать процент готовности`;
- если настройка отключена, поле прогресса не показывается в карточке задачи.

## UX

Основной способ изменения прогресса — ползунок.

- шаг по умолчанию: `5%`;
- администратор может выбрать системный шаг `1%`, `5%` или `10%`;
- точный ввод числа допускается как дополнительный способ;
- ползунок должен быть удобен для мыши, клавиатуры и сенсорного экрана;
- рядом отображаются числовое значение и понятная текстовая интерпретация состояния.

Пример ориентировочных подписей:

- `0%` — не начато;
- `25%` — начато;
- `50%` — половина работы;
- `75%` — почти готово;
- `100%` — выполнено.

Точные тексты должны локализоваться для `ru`, `uk`, `en`.

## Автоматизация при закрытии

При переводе задачи в закрывающий статус, если готовность меньше `100%`, система предлагает установить `100%`.

- вариант по умолчанию: подтвердить установку `100%`;
- пользователь может отказаться;
- автоматическое изменение также фиксируется в истории.

## Будущие независимые показатели

Архитектура не должна считать ручной процент единственным источником прогресса. После MVP могут быть добавлены независимые показатели:

- прогресс по подзадачам;
- прогресс по чек-листу;
- соотношение затраченного и оценочного времени.

Эти показатели не заменяют ручную оценку исполнителя и отображаются отдельно.

## Почему процент не зависит от статуса

Автоматическое вычисление по статусу отклонено: возврат задачи из проверки в работу не должен искусственно уменьшать её фактическую готовность.
