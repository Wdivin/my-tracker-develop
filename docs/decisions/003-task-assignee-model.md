# Решение 003. Один основной исполнитель задачи

Статус: утверждено.

## Решение

В MVP у каждой задачи может быть только один основной исполнитель.

Совместная работа организуется через:

- подзадачи;
- наблюдателей;
- комментарии;
- связи между задачами;
- учёт времени нескольких пользователей.

Поле «соисполнители» не входит в MVP. При необходимости его можно добавить позже как отдельную функцию или пользовательское поле.

## Причины

Один основной исполнитель:

- сохраняет персональную ответственность;
- упрощает фильтры, отчёты и уведомления;
- делает мобильный интерфейс понятнее;
- снижает неоднозначность при завершении задачи;
- не мешает нескольким людям участвовать в работе через подзадачи и журнал активности.

## Ограничения

- наблюдатели не считаются исполнителями;
- автор задачи не обязан быть исполнителем;
- учёт времени допускается для разных пользователей независимо от основного исполнителя;
- права на назначение и изменение исполнителя определяются ролями и workflow.
