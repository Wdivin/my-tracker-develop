# 017. Администрирование, безопасность и интеграция с доменом

## Статус

Утверждено.

## Пользователи

- Пользователи создаются системным администратором либо поступают из доменной интеграции.
- Публичной регистрации нет.
- Пользователь с историей деактивируется, а не удаляется.
- Деактивация завершает активные сессии, отзывает токены и исключает пользователя из новых назначений.
- Задачи деактивированного пользователя не переназначаются автоматически.
- Повторная активация восстанавливает доступ, но не оживляет ранее отозванные токены.

## Системные администраторы

- Системный администратор может управлять другими системными администраторами, включая их деактивацию.
- При этом администратор не редактирует аудит, не видит секреты и не обходит workflow.
- Критические действия требуют явного подтверждения и фиксируются в аудите.

## Аутентификация и безопасность

- Вход по локальной учётной записи поддерживается.
- Двухфакторная аутентификация в MVP не входит.
- Поддерживаются безопасное хеширование паролей, приглашения, сброс пароля, ограничение попыток входа и управление активными сессиями.
- Администратор не задаёт и не видит пароль пользователя.

## Интеграция с доменом

- В MVP нужна интеграция с корпоративным доменом, ориентированная на Active Directory/LDAP.
- Она должна позволять аутентифицировать доменных пользователей и создавать либо обновлять их локальные профили в трекере.
- Домен не заменяет локальную модель прав: роли, проекты, группы и workflow остаются внутри трекера.
- Сопоставление пользователя выполняется по стабильному доменному идентификатору и/или уникальному email согласно выбранной реализации.
- Деактивированный или недоступный в домене пользователь не удаляется из истории трекера.
- Должна сохраняться локальная аварийная учётная запись системного администратора, не зависящая от доступности домена.
- Отдельный административный REST API для создания и синхронизации пользователей в MVP не требуется.

## Группы

- Группы используются только для прав и назначения ролей.
- Группа не может быть исполнителем, автором, наблюдателем или владельцем записи времени.
- Доменная интеграция может использовать доменные группы как источник сопоставления, но конечные права применяются внутри трекера.

## Локализация и часовые пояса

- Интерфейс поддерживает русский и украинский языки.
- Пользователь выбирает локаль и часовой пояс.
- Даты без времени не смещаются между часовыми поясами.

## Брендирование

- Администратор может загружать, заменять и удалять логотип системы.
- Изменение логотипа не требует доработки кода.
- Для логотипа действуют ограничения формата и размера.

## Почта и фоновые процессы

- SMTP настраивается из административного интерфейса.
- Есть тестовая отправка письма.
- Очереди и планировщик имеют базовую диагностику.
- Основные пользовательские действия не откатываются из-за временной недоступности почты.

## Системные настройки

Настройки группируются по разделам:

- основные;
- пользователи и безопасность;
- доменная интеграция;
- почта и уведомления;
- файлы;
- API;
- история и обслуживание.

Критические изменения требуют подтверждения и показывают последствия.

## Обслуживание

- Резервное копирование и восстановление документируются, но не управляются через web-интерфейс.
- Обновления выполняются через штатный процесс развёртывания.
- Есть экран состояния системы без раскрытия секретов.
- Есть административная корзина с восстановлением и физическим удалением по правилам аудита.
- Начальная установка создаёт первый локальный административный аккаунт и базовую конфигурацию.

## Не входит в MVP

- TOTP/SMS 2FA;
- отдельный административный API пользователей;
- SAML и OAuth-вход;
- публичная регистрация;
- мультитенантность;
- автоматическое обновление из интерфейса;
- встроенное облачное резервное копирование;
- управление сервером и контейнерами из приложения.
