# 010. Списки задач, фильтры, сохранённые представления и «Моя работа»

## Статус

Утверждено.

## Основные экраны

В MVP входят четыре основных экрана работы с задачами:

1. «Моя работа»;
2. список задач проекта;
3. доска проекта;
4. глобальный список всех доступных пользователю задач.

Отдельный пункт навигации «Недавно просмотренные» существует вне раздела «Моя работа».

## «Моя работа»

Раздел содержит следующие секции:

- требуют внимания;
- назначены мне;
- созданные мной;
- наблюдаемые.

Секция «Ожидают других» не создаётся.

«Созданные мной» показывает все открытые задачи, созданные пользователем, независимо от того, является ли он их исполнителем.

Причина попадания задачи в «Требуют внимания» всегда отображается явно. Допустимые причины основаны на прозрачных правилах, например:

- задача просрочена;
- срок сегодня;
- новое назначение ещё не просмотрено;
- задача возвращена в работу и ещё не просмотрена;
- есть непрочитанное упоминание;
- задача заблокирована.

Скрытых рейтингов важности и автоматической сортировки по неочевидным правилам нет.

## Представление списка

На desktop задачи показываются таблицей, на мобильных устройствах — карточками.

Базовые колонки desktop-списка:

- номер;
- название;
- проект;
- тип;
- статус;
- исполнитель;
- приоритет;
- готовность;
- срок;
- дата обновления.

Пользователь может:

- включать и выключать колонки;
- менять порядок и ширину колонок;
- сортировать;
- закреплять основные колонки;
- выбирать плотность строк.

Состояние хранится внутри конкретного сохранённого представления.

## Быстрое редактирование

На desktop допускается inline-редактирование:

- доступного workflow-перехода;
- исполнителя;
- приоритета;
- срока;
- готовности;
- названия.

Через inline-редактирование нельзя менять:

- проект;
- тип задачи;
- родительскую задачу;
- сложные пользовательские поля;
- состояние удаления.

На мобильных устройствах действия выполняются через компактное меню или нижнюю панель, а не через сжатую таблицу.

## Фильтры

Фильтр строится как набор условий «поле + оператор + значение».

В MVP все отдельные условия объединяются через `И`. Несколько значений внутри одного условия объединяются через `ИЛИ`.

Пример:

- статус входит в «Новая» или «В работе»;
- исполнитель равен текущему пользователю;
- срок до указанной даты.

Произвольные вложенные группы `И/ИЛИ` в MVP не входят.

Поддерживаются операторы для:

- текста;
- справочников;
- дат;
- чисел и готовности;
- пользователей;
- пустых и заполненных значений.

Активные условия всегда видны пользователю.

## Быстрые фильтры и поиск

Над списком доступны быстрые фильтры:

- только открытые;
- мои;
- без исполнителя;
- просроченные;
- срок сегодня;
- недавно обновлённые;
- с непрочитанными событиями.

Быстрый поиск в текущем списке ищет по номеру и названию задачи.

Поиск по описанию, комментариям и текстовым пользовательским полям относится к глобальному полнотекстовому поиску.

## Сортировка и группировка

Поддерживается последовательная сортировка по нескольким полям.

Поддерживается одна визуальная группировка, например по:

- статусу;
- исполнителю;
- проекту;
- типу;
- приоритету;
- сроку по периодам.

Группировка не изменяет состав выборки.

Система не должна скрыто поднимать просроченные или иные задачи вверх без выбранной пользователем сортировки.

## Сохранённые представления

Сохранённое представление включает:

- область данных;
- фильтры;
- сортировку;
- одну группировку;
- набор, порядок и ширину колонок;
- плотность строк;
- включение задач подпроектов;
- режим списка или доски, если применимо.

Представления бывают:

- личные;
- проектные.

Любой обычный пользователь может создавать личные представления без отдельного права.

Создание и изменение проектных представлений требует соответствующего проектного права.

Изменения открытого проектного представления сначала являются локальными. Оно не перезаписывается автоматически. Пользователь может:

- сохранить изменения при наличии права;
- сохранить копию как личное представление;
- сбросить локальные изменения.

Пользователь может осознанно закреплять представления в избранном и менять их порядок.

## Массовые операции

Список поддерживает массовый выбор и операции:

- смена исполнителя;
- изменение приоритета;
- изменение срока;
- изменение готовности;
- изменение тегов;
- workflow-переход;
- перенос в другой проект;
- изменение типа;
- мягкое удаление.

Каждая выбранная задача проверяется отдельно. Частичный результат допустим и всегда показывается подробно по каждой неуспешной задаче.

Можно выбрать все результаты текущего фильтра, а не только текущую страницу. Большие операции требуют отдельного подтверждения.

## Доска

Доска строится по статусам workflow.

Перетаскивание карточки выполняет реальный workflow-переход. Если переход требует комментария, обязательного поля или подтверждения предупреждения, открывается форма завершения перехода.

Drag-and-drop не является единственным способом. Доступные действия также есть в меню карточки.

Общая доска проекта и подпроектов доступна только тогда, когда workflow используют одинаковый набор статусов в одинаковом порядке. Совпадение по названию не считается совместимостью.

## Пагинация и возврат к списку

Используется серверная пагинация с разумными размерами страницы, например 25, 50 и 100 задач.

Бесконечная прокрутка не используется как основная модель.

При возврате из карточки задачи сохраняются:

- текущая страница;
- позиция прокрутки;
- фильтры;
- сортировка;
- выбранное представление.

## Экспорт

CSV-экспорт текущего представления входит в MVP, но доступен только пользователям с отдельным правом экспорта.

Экспорт учитывает:

- доступ пользователя;
- активные фильтры;
- выбранные колонки;
- текущую сортировку;
- локализованные заголовки;
- однозначный формат дат.

## Что не входит в MVP

- секция «Ожидают других»;
- вложенные логические группы фильтров;
- многоуровневая группировка;
- бесконечная прокрутка как основная навигация;
- XLSX-экспорт;
- скрытая автоматическая сортировка по важности.
