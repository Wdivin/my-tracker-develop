# My Tracker 1.0 — журнал сравнительной приёмки

## Назначение

Это нейтральный, версионируемый реестр результатов сравнительной приёмки независимых реализаций. Он не является частью исходников кандидатов и не заменяет их собственные отчёты.

## Правила статусов

- **Подтверждён live** — воспроизведён на изолированном стенде.
- **Подтверждён по коду** — независимо установлен чтением конкретного кода.
- **Проверить live** — найден review; требуется воспроизведение на стенде.
- **Исправлен** — исправление принято только после повторного сценария.
- В реестр не записываются пароли, токены, cookie и иные секреты.

## Обязательный критерий исправления: невозможное действие

Для любого пользовательского действия, которое нельзя выполнить из-за незаполненных полей, отсутствующей настройки, прав, связей или иного предусловия:

1. интерфейс **не должен** молча оставаться без изменений;
2. пользователь должен увидеть понятное сообщение на выбранной локали с конкретной причиной, например: «Не выбран тип задачи», «Не настроен workflow для проекта», «Заполните название»;
3. сообщение должно указывать, что именно требуется заполнить/настроить и, где возможно, давать переход к месту исправления;
4. после устранения причины действие должно выполняться без повторного поиска скрытых настроек.

Исправление принимается только после live-проверки как отрицательного сценария (предусловие отсутствует) и положительного сценария (оно устранено).
