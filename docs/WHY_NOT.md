# Осознанно отвергнутые решения

Документ объясняет, почему некоторые внешне привлекательные решения не входят в текущую концепцию продукта.

## N-001. Неограниченная глубина подпроектов

**Почему кажется полезной:** максимальная свобода построения структуры.

**Почему отказались:** ухудшает навигацию, отчёты, права и мобильный интерфейс. Для целевой аудитории достаточно трёх уровней.

**Когда пересмотреть:** при подтверждённой потребности крупных внедрений.

**Статус:** отклонено для MVP.

## N-002. Автоматический процент готовности по статусу

**Почему кажется полезным:** не требует ручного обновления.

**Почему отказались:** возврат задачи из проверки в работу искусственно уменьшает прогресс, хотя фактическая готовность может почти не измениться.

**Когда пересмотреть:** не планируется как основной механизм; могут добавляться отдельные автоматические показатели.

**Статус:** отклонено.

## N-003. Несколько основных исполнителей задачи

**Почему кажется полезным:** отражает совместную работу.

**Почему отказались:** размывает персональную ответственность, усложняет уведомления, отчётность и завершение задачи.

**Альтернатива:** один основной исполнитель, подзадачи, наблюдатели, комментарии и связи.

**Статус:** отклонено.

## N-004. Полный offline-first режим

**Почему кажется полезным:** работа без подключения к сети.

**Почему отказались:** значительно увеличивает стоимость и риск MVP, требует синхронизации и разрешения конфликтов, а подтверждённой потребности нет.

**Когда пересмотреть:** только при появлении реальных полевых сценариев без стабильной сети.

**Статус:** исключено из MVP.

## N-005. Частичное смешанное наследование конфигурации подпроектов

**Почему кажется полезным:** позволяет отдельно наследовать workflow, поля, роли и представления.

**Почему отказались:** источник правил становится неочевидным, администрирование превращается в расследование.

**Альтернатива:** подпроект либо полностью наследует конфигурацию, либо использует собственную.

**Статус:** отклонено.

## N-006. Публичная самостоятельная регистрация

**Почему кажется полезной:** упрощает подключение новых пользователей.

**Почему отказались:** продукт ориентирован на внутреннюю работу организации и требует управляемого доступа.

**Альтернатива:** создание или приглашение пользователя уполномоченным сотрудником.

**Статус:** исключено из MVP.
